Entity Relationship Diagram

Proof-of-Delivery Verification System — Supabase PostgreSQL + Object Storage

Data Integrity

  • • SHA-256 image_hash computed client-side before upload
  • • Three timestamp layers: captured_at (GPS), DateTimeOriginal (EXIF), created_at (server)
  • • Cross-referencing timestamps detects clock manipulation
  • • accuracy_status tracks whether GPS met threshold or was overridden

Storage Design

  • • Two isolated buckets: production vs field-test
  • • Images uploaded as binary (base64 → Uint8Array)
  • • EXIF GPS tags burned into JPEG before upload
  • • Public read URLs stored as image_url in records
  • • Filename convention: {context}_{unix_ms}.jpg

Field Test Superset

  • • field_test_records contains all delivery_records columns
  • • Additional: model diagnostics (confidence, detection, override)
  • • Ground truth annotation for confusion matrix analysis
  • • 5 timing metrics (barcode, GPS, capture, EXIF, upload)
  • • Device state: battery level, GPS speed, heading
